## ✅ Cách xử lý triệt để: đặt `client_encoding` = 'UTF8'

### ✳️ **Cách 1: Thêm lệnh ngay sau khi đăng nhập**

Khi đã vào `psql`, gõ:

```sql
SET client_encoding = 'UTF8';
```

Rồi **chạy lại câu lệnh SELECT**:

```sql
SELECT * FROM users WHERE firebase_uid = 'Ct9lxHivV9eOTwyza6ZrkReNH7v2';
```

---

### ✳️ **Cách 2: Thêm biến môi trường khi khởi động**

Trên dòng lệnh CMD (chưa vào `psql`), bạn gõ:

```bash
set PGCLIENTENCODING=utf8
```

Rồi đăng nhập lại:

```bash
psql -U admin -d mental_health_app
```

---

### 💡 Vì sao có lỗi này?

* Dữ liệu trong bảng có **ký tự Unicode (Tiếng Việt)** như “ệ”.
* `psql` mặc định lấy mã hóa từ môi trường hệ thống (Windows thường là `WIN1252`), không khớp với `UTF8` của PostgreSQL.
